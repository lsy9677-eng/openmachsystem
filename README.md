# 230MATCH 본선 운영 V2 — 독립 개발 패키지

## 목적

기존 `dev.html`의 누적 본선 패치와 완전히 분리된 독립 개발 페이지입니다.

실행 파일:

- `main-v2.html`

본선 데이터는 하나의 상태 객체만 사용합니다.

- 대진표
- 본선 경기 현황
- 코트 현황
- 공용대기
- 결과와 다음 라운드

모든 화면이 같은 데이터만 읽습니다.

## GitHub 업로드

압축을 풀고 저장소 루트에 구조 그대로 올립니다.

```text
main-v2.html
assets/css/main-v2.css
assets/js/main-v2/
data/
docs/
backup/
```

GitHub Pages 또는 Cloudflare Pages에서 다음 주소로 확인합니다.

```text
https://사이트주소/main-v2.html?v=1
```

## 현재 구현 완료

- 64팀/32팀 가지형 대진표
- 전체 라운드 고정 생성
- 실제 스코어 저장
- 승자 다음 라운드 자동 진출
- 현재 라운드 본선 현황
- 코트별 시합중 1 + 대기1 1
- 나머지 공용대기
- 경기 종료 시 대기1 → 시합중
- 공용대기 → 대기1
- 공용대기에서 시합중으로 직접 이동 금지
- 균등 배정 순서
- 자동배정 ON/OFF
- 로컬 저장
- JSON 백업/복원

## 중요한 제한

이 패키지는 기존 운영 Firebase에 아직 직접 연결하지 않습니다.

먼저 독립 페이지에서 로직을 검증한 뒤, 기존 앱의 실제 데이터 구조를 읽고 쓰는
`firebase-repository.js` 또는 `legacy-adapter.js`를 추가 연결해야 합니다.

운영 데이터에 바로 연결하지 않은 이유는 기존 본선 함수와 충돌하거나 실제 대회 데이터를 손상시키지 않기 위해서입니다.

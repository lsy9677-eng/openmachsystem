# 기존 앱 연결 계획

## 1단계 — 독립 엔진 검증
현재 패키지의 대진·결과·큐 규칙을 데모 데이터로 검증합니다.

## 2단계 — 읽기 전용 어댑터
기존 앱의 예선 순위와 코트 목록만 읽어 V2 상태로 변환합니다.
운영 Firebase에는 쓰지 않습니다.

## 3단계 — 별도 테스트 경로
Firebase에 `mainV2Test/{tournament}/{division}` 같은 별도 경로를 사용합니다.

## 4단계 — 결과 양방향 연결
V2에서 결과 저장 시 기존 코트 카드가 동일 결과를 읽도록 연결합니다.

## 5단계 — 운영 전환
구 본선 렌더러와 구 자동배정을 제거한 뒤 V2를 운영 경로로 승격합니다.

## 금지 사항
- 기존 본선 함수 위에 wrapper를 반복 추가
- DOM을 주기적으로 숨기는 방식
- 공용대기에서 시합중으로 직접 이동
- 화면별로 라운드·승자를 별도 계산
- 운영 데이터 경로에서 바로 실험
